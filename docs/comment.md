# 주석 작성 규칙

## 1. 주석 기본 원칙
- 주석은 **왜(Why)**와 **어떻게(How)**를 설명하는 데 집중
- 코드 자체로 의미가 충분하면 주석 생략
- 변경 시 반드시 주석도 함께 업데이트
- 불필요한 주석(코드 설명이 아닌 코드 반복)은 작성하지 않음

---

## 2. 주석 종류

### 2.1 한 줄 주석 (Single-line comment)
- `//` 사용
- 코드 위에 배치하여 설명
```dart
// 사용자 프로필 데이터를 불러온다
final profile = await api.getUserProfile();
```

### 2.2 여러 줄 주석 (Multi-line comment)
- `/* */` 사용
- 긴 설명이나 블록 단위 비활성화 시 사용
```dart
/* 
 * 이 함수는 서버에서 사용자 정보를 가져온 뒤
 * 로컬 캐시에 저장하고, UI를 갱신한다.
 */
```

### 2.3 문서화 주석 (Documentation comment)
- `///` 사용 (dartdoc에서 인식)
- 클래스, 메서드, 속성 설명에 사용
```dart
/// 사용자 정보를 관리하는 컨트롤러
///
/// 서버에서 데이터를 가져오고 상태를 관리한다.
class UserController extends GetxController {
  /// 현재 로그인한 사용자 이름
  final userName = ''.obs;

  /// 사용자 프로필 불러오기
  Future<void> fetchUserProfile() async { ... }
}
```

## 3. 위치별 주석 규칙

### 3.1 클래스 주석
- 클래스 정의 바로 위에 문서화 주석(`///`) 작성
- 클래스의 목적과 주요 기능 설명
```dart
/// 홈 화면의 상태를 관리하는 컨트롤러
///
/// 카운터 상태와 증감 로직을 포함한다.
class HomeController extends GetxController { ... }
```

### 3.2 메서드 주석
- 메서드 정의 바로 위에 작성
- 매개변수, 반환값, 예외 상황 설명
```dart
/// 카운터를 1 증가시킨다.
///
/// 예외: count 값이 100을 초과하면 증가하지 않는다.
void increment() { ... }
```

### 3.3 변수 주석
- 의미가 명확하지 않은 변수는 선언 위에 작성
```dart
// 사용자 권한 목록
final permissions = <String>[].obs;
```

### 3.4 섹션 구분 주석
- 파일 내에서 기능별로 구분
```dart
// ====== Private Methods ======

// ====== API Calls ======
```

---

## 4. TODO / FIXME 태그
- 추후 작업이나 버그 수정을 명시
```dart
// TODO: 다국어 지원 추가
// FIXME: 로그인 실패 시 에러 메시지가 표시되지 않음
```

---

## 5. 주석 작성 스타일
- 문장은 간결하게 작성, 필요 시 bullet point 활용
- 한글 주석 사용 가능, 팀 내 표준 언어 준수
- 첫 글자는 대문자, 끝은 마침표 생략 가능

## 📌 요약 규칙
| 구분           | 기호   | 용도                                |
|----------------|--------|-------------------------------------|
| 한 줄 주석     | //     | 간단한 코드 설명                     |
| 여러 줄 주석   | /* */  | 긴 설명, 코드 블록 비활성화           |
| 문서화 주석    | ///    | 클래스, 메서드, 속성 설명 (dartdoc)   |
| 섹션 구분      | // ====| 파일 내 구역 구분                    |
| TODO/FIXME     | //     | 향후 작업, 버그 표시                  |
